# Dasar Teori

Git merupakan sistem kontrol versi terdistribusi yang diciptakan oleh Linus Torvalds pada tahun 2005 untuk mendukung pengembangan kernel Linux. Berbeda dengan sistem terpusat seperti SVN, Git memungkinkan setiap pengembang memiliki salinan lengkap repositori di mesin lokal, sehingga proses pengerjaan tidak bergantung pada jaringan dan berjalan sangat cepat melalui mekanisme snapshot di setiap commit. Kekuatan utama Git terletak pada konsep repository sebagai penyimpan riwayat, serta fitur branching yang memungkinkan pengembangan fitur dilakukan secara paralel tanpa mengganggu kode utama. Dengan identitas unik berbasis hash SHA-1 pada setiap titik simpan, Git memastikan integritas data dan mempermudah proses penggabungan kode (merge) secara otomatis.
Dalam ekosistem pengembangan modern, Git telah menjadi standar industri yang dipadukan dengan platform seperti GitHub, GitLab, atau Bitbucket untuk memfasilitasi kolaborasi tim yang lebih terstruktur. Melalui alur kerja seperti Git Flow, tim pengembang dapat mengelola perubahan kode dengan lebih aman lewat mekanisme pull request dan code review sebelum disatukan ke proyek utama. Selain itu, fleksibilitas Git diperkuat dengan perintah tingkat lanjut seperti rebase, cherry-pick, dan stash yang memberikan kendali penuh bagi pengembang dalam mengelola riwayat perubahan. Kombinasi antara kemandirian lokal dan kemampuan kolaborasi global inilah yang membuat Git tetap relevan dan dominan hingga saat ini.
