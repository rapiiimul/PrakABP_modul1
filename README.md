# Dasar Teori

Git adalah sistem kontrol versi terdistribusi (distributed version control system) yang dirancang oleh Linus Torvalds pada tahun 2005 untuk mendukung pengembangan kernel Linux. Berbeda dengan sistem kontrol versi terpusat seperti SVN, Git menyimpan salinan lengkap repositori di setiap mesin pengembang, sehingga hampir semua operasi dapat dilakukan secara lokal tanpa bergantung pada koneksi jaringan. Git bekerja dengan merekam snapshot dari seluruh kondisi proyek setiap kali perubahan disimpan (commit), bukan hanya mencatat perbedaan antar file. Pendekatan ini membuat Git sangat cepat dan efisien dalam mengelola riwayat perubahan kode sumber.
Konsep inti dalam Git meliputi repository, branch, commit, dan merge. Repository adalah tempat penyimpanan seluruh riwayat proyek, yang terdiri dari direktori .git berisi metadata dan objek data. Branch memungkinkan pengembang bekerja pada fitur atau perbaikan secara terpisah tanpa mengganggu kode utama (main branch), sehingga mendukung pengembangan paralel yang terstruktur. Setiap commit merepresentasikan titik simpan yang unik, diidentifikasi oleh hash SHA-1, dan memuat informasi tentang siapa yang membuat perubahan, kapan, dan apa yang berubah. Proses merge kemudian digunakan untuk menggabungkan perubahan dari satu branch ke branch lainnya, dengan Git secara otomatis menyelesaikan konflik sederhana.
Dalam praktik pengembangan perangkat lunak modern, Git dipadukan dengan layanan remote repository seperti GitHub, GitLab, atau Bitbucket untuk memfasilitasi kolaborasi tim. Alur kerja umum seperti Git Flow atau GitHub Flow memberikan panduan terstruktur tentang cara membuat branch, melakukan pull request, dan melakukan code review sebelum perubahan digabungkan ke kode utama. Git juga mendukung berbagai perintah tingkat lanjut seperti rebase, cherry-pick, dan stash yang memberikan fleksibilitas tinggi dalam mengelola riwayat proyek. Dengan fitur-fitur tersebut, Git telah menjadi standar de facto dalam industri pengembangan perangkat lunak di seluruh dunia.
